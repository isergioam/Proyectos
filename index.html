Requiere Base de Datos

Este proyecto utiliza MySQL. Necesitas levantar una base de datos local y configurar las variables de entorno.

Pasos para arrancar el proyecto:

1
Abrir terminal en el directorio raíz

Asegúrate de estar en la carpeta principal del repositorio clonado.

2
Instalar todas las dependencias

Hemos configurado scripts automáticos en el package.json principal para que no tengas que ir carpeta por carpeta.

npm run install:cacherunner
3
Configurar base de datos MySQL (Si aplica)

Crea una base de datos MySQL en tu servidor local e importa el archivo SQL del proyecto. Configura el archivo .env en el directorio backend del proyecto con tus credenciales.

# Archivo de esquema:\n/FitQuest/backend/schema.sql
4
Arrancar frontend y backend concurrentemente

Ejecuta el siguiente comando para levantar el servidor de desarrollo del Frontend (Vite) y el Backend (Node/Express) al mismo tiempo:

npm run dev:cacherunner

Para que la gente pueda ejecutar este proyecto desde sus casas de forma 100% online y gratuita, debes desplegar el frontend y el backend por separado:

1. Desplegar el Frontend (Cliente React)

GitHub Pages / Vercel / Netlify

Despliega el frontend estático. Dado que son aplicaciones SPA con React Router, la opción más rápida y recomendada es Vercel o Netlify, que admiten subcarpetas de monorepos de forma nativa.

  • Crea una cuenta en Vercel y vincula tu GitHub.
  • Crea un "New Project" y selecciona este repositorio.
  • En Root Directory, selecciona la carpeta del frontend (ej: CacheRunner/frontend).
  • En Variables de Entorno, añade VITE_API_URL apuntando a la URL del backend desplegado.

2. Desplegar el Backend (Servidor Node)

Render / Koyeb / Railway

El backend necesita un servidor Node.js activo y, en algunos casos, una base de datos MySQL en la nube (como la capa gratuita de Aiven o PlanetScale, o Supabase/Neon si migras a PostgreSQL).

  • Despliega en Render creando una "Web Service" vinculada a este repositorio.
  • Configura el Root Directory al backend (ej: CacheRunner/backend).
  • Comando de inicio: npm install && npm start.
  • Si el proyecto requiere base de datos, añade las variables de entorno de conexión (DB_HOST, DB_USER, DB_PASSWORD, DB_NAME, DB_PORT) correspondientes a tu base de datos remota.

Nota importante sobre las URLs de la API: Para que el frontend pueda hablar con el backend en producción, debes cambiar las URLs hardcodeadas de http://localhost:3000 a import.meta.env.VITE_API_URL || 'http://localhost:3000' en tu código (ej: en el archivo services/api.js).
